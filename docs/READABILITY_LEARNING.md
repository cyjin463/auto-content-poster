# 가독성 학습 시스템

## 📖 개요

최근 4건의 포스팅(한글 2건, 영문 2건)을 분석하여 가독성을 평가하고, 개선 사항을 다음 포스팅에 반영하는 자가 학습 시스템입니다.

## 🔍 가독성 평가 항목

### 1. 문단 길이
- 적절한 길이: 3-5문장
- 문제: 너무 길거나 짧은 문단
- 개선: 긴 문단을 나누고, 짧은 문단은 적절히 연결

### 2. 문장 길이
- 적절한 길이: 한 문장 20-30단어 이내
- 문제: 너무 긴 문장으로 읽기 어려움
- 개선: 복잡한 문장을 간단한 문장으로 분리

### 3. 소제목 사용
- 적절한 개수: 본문에 3-4개 이상
- 문제: 소제목이 부족하거나 과도함
- 개선: 내용을 논리적으로 구분하는 소제목 추가

### 4. 리스트 사용
- 마크다운 리스트 형식: `1.`, `2.`, `3.` 또는 `-`, `-`, `-`
- 문제: 나열할 내용을 문단으로 작성
- 개선: 특징, 사례, 원리 등을 리스트 형식으로 작성

### 5. 볼드체 사용
- **볼드체** 형식: `**텍스트**`
- 문제: 중요한 키워드나 개념이 강조되지 않음
- 개선: 중요한 정보는 볼드체로 강조

### 6. 문단 구분
- 빈 줄: 문단 사이, 소제목 다음에 빈 줄 필수
- 문제: 형식 없이 통으로 작성됨
- 개선: 모든 문단 사이 빈 줄 추가

### 7. 전체적인 구조
- 명확한 흐름: 서론 → 본론 → 결론
- 문제: 구조가 불명확하거나 흐름이 어색함
- 개선: 논리적인 구조와 자연스러운 전환

## 🔄 학습 프로세스

1. **캐시에서 포스트 조회**
   - 한글 최근 2건
   - 영문 최근 2건
   - Notion 참조 없이 로컬 DB에서 조회

2. **가독성 분석**
   - 위 7가지 항목 평가
   - 문제점 도출
   - 개선 제안 생성

3. **분석 결과 반영**
   - 콘텐츠 생성 프롬프트에 포함
   - 다음 포스팅에 자동 반영

## ✅ 효과

- 지속적인 가독성 개선
- 독자 친화적인 글쓰기
- 자연스럽고 읽기 쉬운 콘텐츠
- 구조화된 정보 제공
